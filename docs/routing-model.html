Coding-Agent Routing — Canonical Model

One expensive brain, cheap fast hands, and a machine that keeps it that way.

Northstar — hands vs brain

Optimize for keeping Fable usage safe without sacrificing intelligence. Fable weekly capacity is a hard scarce resource (the pool has hit 100% of its 5-hour limit with Fable-out 60–75% on two accounts). The brain — Fable — decides, architects, verifies, and writes the artifacts that need full context (plans, PRDs, specs, docs like this one). The hands — cheap seats — do volume: reads, mechanical edits, retries, builds, test runs. Parallel fan-out is encouraged; the system gates economics, never concurrency.

The session map (the instrument everything reads)

Every Anthropic-path request through agent-lb persists its Claude Code session identity (metadata.user_id JSON → X-Claude-Code-Session-Id header → null, never synthesized), and the Sol bridge prefers real client identity over synthetic turn ids at log-write — so subagent seat traffic lands under its coordinator session, across retries.

SurfaceWhat it answers
GET /api/sessionswho's active: models, requests, tokens (cached split), cost, errors, sparklines
GET /api/sessions/{id}/analyticswindowed per-seat totals (model + reasoning effort), bucketed timeline, latency & tokens-per-request histograms — series sums equal session totals
GET /s/{8-char prefix}short link → dashboard deep-link (/sessions?session=…) — lives in every status line
Dashboard → Sessionslist + full-width analytics view: stat tiles, stacked timeline, seat donut/table, histograms

Known limit: the LB sees seats (model+effort), not agent instances — true "N subagents" counts need client-side transcript enrichment (fleet-audit analyzer is the prior art; backlogged).
